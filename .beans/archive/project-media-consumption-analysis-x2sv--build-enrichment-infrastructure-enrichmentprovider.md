### Key Decisions
1. **Enrichment fully decoupled from sync** — SyncService no longer runs enrichment inline. Items are stored with status='pending' and the worker picks them up.
2. **Worker uses `workerTestMediaStore` wrapper** for tests — allows overriding just the claim/update methods while reusing the existing mock.
3. **Tags merged by ExternalID** — coordinator merges API provider results back onto items using ExternalID as the join key.
4. **LLM tag results stored in `_llm_tag_result` metadata key** — allows the worker to distinguish LLM tags (with confidence) from API tags (authoritative) during persistence.
