# AGENTS.md - Specto

## Project Overview

Specto is a Go web application for tracking media consumption across platforms.
Built with Chi router, PostgreSQL (pgx), templ templates, and Tailwind CSS.

**Module:** `github.com/justestif/specto` | **Go version:** 1.25+

## Build & Run Commands

All task automation uses [mise](https://mise.jdx.dev/). No Makefile or package.json.

```bash
mise run dev              # Start dev server (air live-reload)
mise run build            # Build binary to bin/app
mise run setup            # Run migrations + templ generate + sqlc generate
mise run templ            # Regenerate templ templates
mise run sqlc             # Regenerate type-safe DB code from SQL
mise run db-migrate       # Run pending database migrations
mise run db-rollback      # Rollback last migration
mise run api-test         # Run httpyac API integration tests
```

### Testing

```bash
go test ./...                              # Run all tests
go test ./internal/core/                   # Run tests in one package
go test ./internal/core/ -run TestSyncFull # Run a single test by name
go test ./internal/core/store/...          # Run all store tests
go test -v ./internal/core/ -run TestSync  # Verbose output, pattern match
go test -count=1 ./...                     # Skip test cache
```

Tests use Go's standard `testing` package. No testify or external frameworks.
Tests are co-located with source files (`*_test.go`).

### Code Generation (must regenerate after changes)

- **SQL queries** (`internal/database/queries.sql`) -> run `mise run sqlc`
- **Templates** (`components/*.templ`) -> run `mise run templ`
- **Never edit** `*_templ.go`, `internal/database/db.go`, `internal/database/models.go`,
  or `internal/database/queries.sql.go` -- these are generated by sqlc/templ.

### Infrastructure

```bash
docker compose up -d      # Start PostgreSQL 16 on localhost:5432
```

DB: `specto_dev`, user/pass: `postgres/postgres`

Required env vars: `DATABASE_URL`, `ENCRYPTION_KEY` (64 hex chars),
`SESSION_SECRET` (32+ bytes), `CSRF_KEY` (exactly 32 bytes), `PORT`.
These are set in `mise.toml` for local dev.

## Project Structure

```
cmd/web/main.go          # Entry point, wiring, routes
internal/
  app/                   # Application layer - Config, dependency wiring
  auth/                  # Authentication service + session management
  core/                  # Domain logic (pure, no DB imports)
    store/               # Store implementations (pgx, crypto)
  database/              # sqlc-generated code + DB connection pool
  handlers/              # HTTP handlers (methods on *Handler)
  middleware/            # HTTP middleware (auth, CSRF)
components/              # Templ UI templates
migrations/              # SQL migration files (up/down pairs)
http/                    # httpyac integration test files
styles/                  # Tailwind CSS source
static/                  # Built CSS output
```

## Code Style

### Formatting & Linting

Use `gofmt` / `goimports`. No custom linter config (no `.golangci.yml`).

### Import Order

Three groups separated by blank lines:

```go
import (
    "context"              // 1. stdlib
    "net/http"

    "github.com/go-chi/chi/v5"  // 2. third-party

    "github.com/justestif/specto/internal/core"  // 3. internal
    customMiddleware "github.com/justestif/specto/internal/middleware"  // aliased if needed
)
```

### Naming Conventions

| Element             | Convention      | Example                                   |
|---------------------|-----------------|-------------------------------------------|
| Packages            | lowercase       | `core`, `store`, `handlers`               |
| Exported types      | PascalCase      | `MediaItem`, `SyncService`                |
| Unexported helpers  | camelCase       | `writeJSON`, `writeError`                 |
| Interfaces          | Descriptive noun| `MediaItemStore`, `Enricher`              |
| Sentinel errors     | `Err` prefix    | `ErrInvalidCredentials`, `ErrEmailTaken`  |
| Error codes         | `Err` prefix    | `ErrAuthExpired`, `ErrRateLimit`          |
| Constants (exported)| PascalCase      | `MediaMusic`, `BucketDay`                |
| Context keys        | typed constant  | `contextKey("user")`                      |
| Constructors        | `New` / `NewX`  | `app.New()`, `handlers.New()`             |
| Test helpers         | camelCase       | `newTestSyncService`, `discardLogger`     |

### Architecture Patterns

- **Dependency injection via constructors** -- no global state, deps wired in `main.go`
- **Interface-driven stores** -- interfaces in `core/`, implementations in `core/store/`
- **Compile-time interface checks**: `var _ Enricher = (*NoOpEnricher)(nil)`
- **Domain types separate from DB models** -- conversions in `store/convert.go`
- **Handlers are methods on `*Handler` struct**, not standalone functions

### Error Handling

- Sentinel errors with `errors.New()` for known failure cases
- Custom error types with `Error()` + `Unwrap()` (see `core/errors.go`)
- Wrap with context: `fmt.Errorf("fetching user: %w", err)`
- Check with `errors.Is()` / `errors.As()`, never direct comparison
- Early return on error, no else after error check
- Non-fatal errors are logged and execution continues (e.g., enrichment failures)

### HTTP Conventions

- Routes: API under `/api/v1/`, static files under `/static/`
- JSON responses: `{"data": {...}}` on success, `{"error": {"code": "...", "message": "..."}}` on failure
- Use `writeJSON()` and `writeError()` helpers in handlers
- Middleware signature: `func(http.Handler) http.Handler`

### Testing Patterns

- Standard `testing` package only
- Mocks use function fields: `mockPlugin{syncFn: func(...) {...}}`
- Table-driven tests for value/boundary checks
- Both internal (`package core`) and external (`package core_test`) test packages used
- Test helpers: `newMockPlugin()`, `discardLogger()`, `newTestSyncService()`
- Inline test data, no fixture files

### Templates & Frontend

- Templ for type-safe Go HTML templates (`.templ` files)
- HTMX for dynamic interactions (CDN-loaded)
- DaisyUI + Tailwind CSS via standalone binary (not npm)
